SQL: достать данные из базы

Навык №1 в любой вакансии аналитика. Язык, которым «спрашивают» базу данных.

⚡ Шпаргалка для заучивания Любой запрос собирается из этих кубиков, всегда в таком порядке: 
SELECT   столбцы    — ЧТО достать (или * — всё)
FROM     таблица    — ОТКУДА
JOIN     таблица2 ON условие  — склеить со второй таблицей
WHERE    условие    — КАКИЕ строки (фильтр)
ORDER BY столбец    — КАК отсортировать
Обязательны только SELECT и FROM. Остальное — по надобности. Регистр не важен: SELECT = select.

Учебная база

Две таблицы. Столбец client_id во вкладах указывает на id клиента — это связь между ними.

SELECT … FROM — что и откуда

SELECT name, city FROM clients

Достать столбцы name и city из таблицы clients. Звёздочка * — все столбцы сразу.

WHERE — фильтр строк

SELECT * FROM clients WHERE age > 30

Оставит только строки, где условие истинно. Операторы сравнения:

ОператорЗначениеПример
=равноcity = 'Москва'
> <больше / меньше (строго)age > 30
>= <=больше/меньше или равноrate >= 12
!=не равноproduct != 'Вклад'
🚫 Ловушка: строго или нет rate > 12 НЕ включает ровно 12. Чтобы «12 и выше» — нужно rate >= 12. Одна чёрточка меняет ответ. Текст в условии — всегда в одинарных кавычках: 'Москва'.

ORDER BY — сортировка

SELECT * FROM deposits WHERE rate > 12 ORDER BY amount DESC

DESC — по убыванию (от большего к меньшему), ASC — по возрастанию (по умолчанию, можно не писать).

JOIN — склеить две таблицы

Во вкладах есть только client_id (номер), а имя лежит в clients. JOIN подтягивает имя к каждому вкладу.

SELECT clients.name, deposits.product, deposits.amount
FROM deposits
JOIN clients ON deposits.client_id = clients.id
  • JOIN clients — присоединить вторую таблицу;
  • ON deposits.client_id = clients.id — мостик: где номер клиента во вкладе = id клиента;
  • таблица.столбец — уточняем, из какой таблицы столбец.
📌 Почему строк ровно столько JOIN даёт одну строку на каждую совпавшую пару. 12 вкладов → 12 строк. У кого 2 вклада — попадёт дважды. У кого вкладов нет — выпадет совсем.

LEFT JOIN и NULL — сохранить всех

Обычный JOIN выкидывает строки без пары. LEFT JOIN оставляет все строки из левой таблицы (той, что до LEFT JOIN), а пустые места заполняет NULL.

SELECT clients.name, deposits.product, deposits.amount
FROM clients
LEFT JOIN deposits ON clients.id = deposits.client_id

Вернёт всех 8 клиентов. У кого нет вклада — в product и amount будет NULL.

💡 NULL — это «пусто», не ноль NULL значит «значения нет». Проверять через = нельзя: WHERE amount = NULL не сработает. Только IS NULL (пусто) и IS NOT NULL (не пусто).

Практическая задача — «кто не купил ни одного продукта»:

SELECT clients.name
FROM clients
LEFT JOIN deposits ON clients.id = deposits.client_id
WHERE deposits.id IS NULL

Ключевые термины

SELECT … FROM — что и откуда WHERE — фильтр строк ORDER BY … DESC/ASC — сортировка JOIN — только совпавшие пары LEFT JOIN — сохранить все левые строки NULL — пусто (IS NULL / IS NOT NULL) > строго, >= с равенством
🎯 Проверь себя завтра (по базе банк.db) 1. Все вклады продукта «Вклад» с суммой больше 100000, по убыванию суммы.
2. Имя клиента и сумма для каждого вклада (JOIN).
3. Клиенты из Москвы старше 30.
4. Кто из клиентов без вкладов (LEFT JOIN + IS NULL).
